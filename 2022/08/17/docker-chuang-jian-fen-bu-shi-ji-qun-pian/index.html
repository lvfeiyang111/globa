redis分片集群

  • 方式一：sh脚本示例
for port in $(seq 7001 7006); 
do 
mkdir -p /mydata/redis/node-${port}/conf 
touch /mydata/redis/node-${port}/conf/redis.conf 
cat << EOF >/mydata/redis/node-${port}/conf/redis.conf 
port ${port} 
cluster-enabled yes 
cluster-config-file nodes.conf 
cluster-node-timeout 5000
cluster-announce-ip  172.31.171.217 
cluster-announce-port ${port}
cluster-announce-bus-port 1${port}
appendonly yes
EOF
docker run -p ${port}:${port} -p 1${port}:1${port} --name redis-${port} --privileged=true 
-v /mydata/redis/node-${port}/data:/data 
-v /mydata/redis/node-${port}/conf/redis.conf:/etc/redis/redis.conf 
-d redis:5.0.7 redis-server /etc/redis/redis.conf; 
done

方式二：手动一个一个创建

首先每个docker创建redis需要一个data文件夹，一个redis.conf文件

集群文件需要配置以下信息 ，这是其中一个节点的配置文件

#在redis.conf文件中写入
port 7001 #端口号
cluster-enabled yes #是否开启集群模式
cluster-config-file nodes.conf #集群的配置文件名称，不需要自己创建，由redis自己维护
cluster-node-timeout 5000 #超时时间
cluster-announce-ip 192.168.159.138 #注册实例的ip
cluster-announce-port 7001 #集群节点的映射端口默认6379
cluster-announce-bus-port 17001 #集群互相通信的端口
appendonly yes

然后在docker中创建此节点（配置文件目录在/mydata/redis/node-7001/目录下data和conf/redis.conf文件）

docker run -p7001:7001 -p17001:17001 --name redis-7001 --privileged -v /mydata/redis/node-7001/data:/data -v /mydata/redis/node-7001/conf/redis.conf:/etc/redis/redis.conf -d redis:5.0.7 redis-server /etc/redis/redis.conf

以此类推，几个集群节点就创建几个

创建完成之后随意进入一个容器内

docker exec -it redis-7001 bash
redis-cli --cluster create 192.168.159.138:7001 192.168.159.138:7002 192.168.159.138:7003 192.168.159.138:7004 192.168.159.138:7005 192.168.159.138:7006 --cluster-replicas 1 #这个参数1是需要几个从节点

之后系统就会给出集群方案列如

如图所示节点7001是一个master，它的从节点是7005 salve

使用 redis-cli 的 cluster 方式进行连接

redis-cli -c -h 192.168.159.138 -p 7006

扩容

因为redis使用的是哈希槽，每个节点大致平均对应着一片槽位，根据key的哈希值来判断存储在哪个节点上，如果进行扩容如何解决原数据槽位变更带来的读取问题

  1. 首先创建新主节点，进入容器内部

    redis-cli --cluster add-node ip:端口 ip：端口
#如果报错，说nodes不为空，就把容器内data目录下的三个配置文件nodes.cnf等删了重启容器在进去执行命令
#第一个ip:端口就是将要作为master的新增节点，第二个ip:端口是原来节点里的领路人，根据这个节点加入集群
redis-cli --cluster check ip:端口 #查看集群信息

  2. 加入节点之后，新节点master默认槽位是0，现在需要给他分配槽位

    redis-cli --cluster reshard ip:新节点端口 #重新分配槽位,随意输入一个集群master几点的ip：端口
#How many slots do you want to move (from 1 to 16384)? 这里是问你怎么分配，可以根据 16384/节点数 来进行分配 ,例如我现在是4台机器输入4096
#what is the receiving node ID ? #分配给谁，这里填新增master节点的id
#Source node #1: all #你要从哪个节点中抽出这些槽位，这里输入all表示所有节点平均分配一点槽位给新节点
    • 这里的重新分配槽位，是每个原主节点抽了4096个没有存放数据的槽位给新加入的主节点。原有数据不发生改变

给新主节点增加一个从节点

启动一个从节点，进入从节点

redis-cli --cluster add-node ip:新slave端口 ip:新master端口 --cluster-slave --cluster-master-id 新主节点id

缩容

先删从机

redis-cli --cluster del-node ip:从机端口 从机id

将要删除的主节点槽号清空，重新分配

redis-cli --cluster reshard ip:端口  #重新分配槽位
#How many slots do you want to move (from 1 to 16384)? 分配多少槽位 4096
#What is the receiving node ID? 谁来接收这些槽位 删除槽位之后多出来的槽位可以随意给别的master进行接收
#Source node 谁来出这些槽位，输入要删除槽位的ID
done			
redis-cli --cluster check ip:端口 #查看集群信息
#要删除的master节点槽位清空后即可删除
redis-cli --cluster del-node ip:要删除的master端口 要删除的master的Id

RabbitMq

首先创建映射文件 

mkdir /mydata/rabbitmq
cd rabbitmq/
mkdir rabbitmq01 rabbitmq02 rabbitmq03

创建docker容器

主：

docker run -d --hostname rabbitmqMaster --name rabbitmqMaster -v /mydata/rabbitmq/rabbitmq-Master:/var/lib/rabbitmq -p 15672:15672 -p 5672:5672 -e RABBITMQ_ERLANG_COOKIE='lfy' rabbitmq:management
  • –hostname 设置容器主机名，方便从节点连接
  • -e RABBITMQ_ERLANG_COOKIE=’lfy’ 节点认证作用，部署集成时 需要同步该值 相当于一个令牌，跟从节点进行通信的令牌

从：

docker run -d --hostname rabbitmqSlave1 --name rabbitmqSlave1 -v /mydata/rabbitmq/rabbitmq-slave1:/var/lib/rabbitmq -p 15673:15672 -p 5673:5672 -e RABBITMQ_ERLANG_COOKIE='lfy' --link rabbitmqMaster:rabbitmqMaster  rabbitmq:management
  • –link rabbitmqMaster:rabbitmqMaster 主节点的主机名，也就是前面指定的–hostname 。 然后用：取了一个别名 还是叫rabbitmqMaster

从2：

docker run -d --hostname rabbitmqSlave2 --name rabbitmqSlave2 -v /mydata/rabbitmq/rabbitmq-slave2:/var/lib/rabbitmq -p 15674:15672 -p 5674:5672 -e RABBITMQ_ERLANG_COOKIE='lfy' --link rabbitmqMaster:rabbitmqMaster --link rabbitmqSlave1:rabbitmqSlave1 rabbitmq:management
  • –link:别忘了把从节点1的主机名也绑定上去

或者不使用link，使用自定义网络的方式。这里省略方式二

常见错误：

  • 如果在web端打开15672提示不是一个安全的建立连接需要进入容器中创建账号
docker exec -it rabbitmqMaster bash
rabbitmqctl list_users #查看用户列表
rabbitmqctl add_user lfy 123321 #添加一个用户
rabbitmqctl set_user_tags lfy administrator #给名字为lfy的用户权限
  • 如果加入集群报错
Clustering node rabbit@rabbitmq2 with rabbit@rabbitmq1
Error:
incompatible_feature_flags 
表示: Rabbitmq各节点版本不一致导致的
  • 默认rabbitmq启动后是磁盘节点，在这个cluster命令下，rabbitmqSlave1和rabbitmqSlave2是内存节点，rabbitmqMaster是磁盘节点。
    如果要使rabbitmqSlave1、rabbitmqSlave2都是磁盘节点，去掉–ram参数即可。在RabbitMQ集群中，必须至少有一个磁盘节点，否则队列元数据无法写入到集群中，当磁盘节点宕掉时，集群将无法写入新的队列元数据信息。
  • 如果想要更改节点类型，可以使用命令rabbitmqctl change_cluster_node_type disc(ram),前提是必须停掉rabbit应用
  • 也有可能是因为在创建容器时，没有指定所有link 所有主节点和从节点的主机名。

加入集群

给主rabbitmq容器恢复出厂设置，如果是新安装的rabbitmq可以省略这一步

docker exec -it rabbitmq01 /bin/bash
rabbitmqctl stop_app #停止rabbitmq
rabbitmqctl reset #进行重置
rabbitmqctl start_app #启动rabbitmq
Exit

进入从节点加入集群

#进入第二个节点
docker exec -it rabbitmq02 /bin/bash
rabbitmqctl stop_app
rabbitmqctl reset #重置 
rabbitmqctl join_cluster --ram rabbit@rabbitmqMaster #变为rabbitmqMaster主机名的从节点，先要stop_app才能加入集群
rabbitmqctl start_app
exit
  • –ram rabbit@主机名 去掉–ram参数就是以磁盘节点的状态加入集群。加上–ram就是以内存节点。集群中必须存在一个磁盘节点

第三个同理

实现镜像集群

进入主节点容器内

docker exec -it rabbitmqMaster bash
rabbitmqctl set_policy -p / ha "^" '{"ha-mode":"all","ha-sync-mode":"automatic"}'
#ha 策略名字   /  表示指定虚拟主机为/     “^”  表示所有队列 如果想指定某个队列 使用“^hello”则表示以hello开头的队列  
# ha-mode 高可用模式   all 所有都是高可用模式   ha-sync-mode：“automatic” 消息自动同步

  • 可以使用 rabbitmqctl list_policies -p /；查看 vhost/下面的所有 policy

  • 在 cluster 中任意节点启用策略，策略会自动同步到集群节点

    rabbitmqctl set_policy-p/ha-all”^”’{“ha-mode”:“all”}’

    策略模式 all 即复制到所有节点，包含新增节点，策略正则表达式为 “^” 表示所有匹配所有队列名称。“^hello”表示只匹配名为 hello 开始的队列
