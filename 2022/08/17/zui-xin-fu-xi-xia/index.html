1.DockerFile

指令 作用
FROM 基础镜像，当前新镜像是基于哪个镜像的，指定一个已经存在的镜像作为模板，第一条必须是FROM
MAINTAINER 镜像维护者的姓名和邮箱地址
RUN 容器构建时需要运行的命令，两种格式，shell格式、exec格式。RUN是在docker build时运行。相当于在容器内运行终端命令
EXPOSE 当前容器对外暴露的端口
WORKDIR 在创建容器后，终端进入容器的默认目录，一个落脚点
USER 指定该镜像以什么样的身份去执行，如果不指定，默认root
ENV 用来在构建镜像的过程中定义环境变量，可以用$+变量名取值
ADD 将宿主机目录下的文件拷贝进镜像且会自动处理URL和解压tar压缩包
COPY 类似ADD，拷贝文件到镜像中
VOLUME 容器数据卷，用于数据保存和持久化工作 相当于 -v
CMD 指定容器启动后要干的事情，可以存在多个cmd指令，但是只有最后一个生效。RUN是在build运行，CMD是在run的时候运行
ENTRYPOINT 跟cmd差不多 但是当指定了ENTRYPOINT之后CMD的含义就发生了变化，不再是直接运行命令，而是将CMD的内容作为参数传递给ENTRYPOINT

ENTRYPOINT

#例如
ENTRYPOINT ["nginx","-c"]
CMD ["/etc/nginx/nginx.cnf"]
#衍生出来的命令是 nginx -c /etc/nginx/nginx.cnf

示例

#基础镜像
FROM centos:7.5 
#姓名和邮箱
MAINTAINER lfy<1316547530@qq.com> 
#定义一个路径变量
ENV MYPATH /usr/local 、
#进入容器默认目录
WORKDIR $MYPATH  
RUN yum -y install vim  #容器build时，下载vim
RUN yum -y install net-tools
RUN yum -y install glibc.1686
RUN mkdir /usr/local/java 
#提交下载jdk到宿主机中，ADD添加进容器目录里
ADD jdk-8u144-linux-x64.tar.gz /usr/local/java 
#配置java环境变量
ENV JAVA_HOME /usr/local/java/jdk1.8.0_171
ENV JRE_HOME $JAVA_HOME/jre
ENV CLASSPATH $JAVA_HOME/lib/dt.jar:JAVA_HOME/lib/tools.jar:$JRE_HOME/lib:$CLASSPATH
ENV PATH $JAVA_HOME/bin:$PATH
EXPOSE 80

部署java微服务实例

FROM openjdk:8
EXPOSE 8080
MAINTAINER lfy
#挂载容器卷，只写一个就是默认挂载到/var/lib/docker目录下
VOLUME /tmp
#把当前目录的app.jar包 添加进容器根目录
ADD app.jar /app.jar
#修改访问时间，可写可不写
RUN bash -c 'touch /app.jar'
ENTRYPOINT ["java","-jar","/app.jar"]

2.虚悬镜像

错版镜像，构建时出现错误会产生虚悬镜像，特点是名字和tag都是。需要及时删除

查看虚悬镜像

docker image ls -f dangling=true

删除虚悬镜像

docker image prune

3.网络

docker network connect #将某个容器连接到一个docker网络
docker network create #创建一个docker局域网络
docker network disconnect #将某个容器退出某个局域网络
docker network inspect #显示某个局域网络信息
docker network ls  #显示所有docker局域网络
docker network prune #删除所有未引用的docker局域网络
docker network rm  #删除docker网络

网络模式

  • **bridge(默认)**：使用–network bridge指定，默认使用docker0
    • 默认为该模式，相当于VMware 中的nat 模式，容器使用独立network Namespace ，并连接到docker0 虚拟网卡。通过docker0 网桥以及iptables nat 表配置与宿主机通信，此模式会为每一个容器分配Network Namespace、设置IP等，并将一个主机上的Docker 容器连接到一个虚拟网桥上。
    • docker0上面的每个veth匹配某个容器实例内部的eth0，两两配对，一一匹配，
  • host：使用–network host指定
    • 容器不会虚拟出自己的网卡，配置主机的IP等，而是使用宿主机的IP和端口
    • docker启动指定–network=host或-net=host，如果还指定了端口，就会出现警告，并且通过-p设置映射不会起到任何作用，端口号会以主机端口号为主，重复时递增
  • none：使用–network none指定
    • 该模式关闭了容器的网络功能
  • container：使用–network container:NAME 或容器id指定
    • 创建的容器不会创建自己的网卡，配置自己的IP，而是和一个指定的容器共享IP，端口范围
    • 当被继承的容器关闭时，继承他的容器将会没有
docker run --name test --privileged=true  -v xxx:xxx -p 8888:8888 -d xxx #默认bridge
docker run --name test --network host --privileged=true -v xxx:xxx -d xxx #host
docker run --name test --network none --privileged=true -v xxx:xxx -d xxx #none
docker run --name test --network container:需要共用网桥的容器名 --privileged=true -v xxx:xxx -d xxx #container

docker 容器的内部网络是有可能会发生改变的。

场景：比如启动了一个容器a，ip为127.0.0.2 这个容器宕机了，在他宕机期间又启动了另外一个容器b，这个容器也被分配成了127.0.0.2。等容器a恢复启动后，docker会改变他的ip。这时候127.0.0.2就成了容器b的ip了。

解决：自定义网络，让容器加入同一个自定义网络。容器IP变动时可以通过服务名直接网络通信而不收影响

  • 现在link已经过时。推荐使用同一自定义网络搭建集群，并且使用服务名直接网络通信。

3.2自定义网络示例

自定义网络本身就维护好了主机名和ip的对应关系（ip和域名都能通）

 # 不指定网络驱动时默认创建的bridge网络
 docker network create test
 # 查看网络内部信息
 docker network inspect test
docker run --name tommcat -net=test -p 8080:8080 -d tomcat:8.5
docker run --name tomcat2 -net=test -p 8081:8080 -d tomcat:8.5
#进入容器测试
docker exec -it tomcat bash
ping tomcat2

  • create可选参数

    --driver bridge            网络模式默认为桥接模式
--subnet   192.168.0.0/16            子网掩码  后16位可为任意值，大概支持 65535 个IP
--gateway 192.168.0.1                网关IP 
# docker network create --driver bridge --subnet 192.168.0.0/16 --gateway 192.168.0.1 test

java通过服务名连接服务说明

spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://mysql:3306/test?useUnicode=true&characterEncoding=UTF-8&serverTimezone=GMT%2B8
    username: root
    password: 123456
  • 通过服务名mysql：端口号。坑：是容器内部服务运行的端口号，不是向外暴露的端口号！
  • 同理，使用nginx反向代理时，在同一个docker自定义网络，也可以使用服务名来代替ip进行反向代理。不过不建议。

4.Docker-compose容器编排

  1. Docker Compose是一个工具，用于定义和运行多容器应用程序的工具；
  2. Docker Compose通过yml文件定义多容器的docker应用；
  3. Docker Compose通过一条命令根据yml文件的定义去创建或管理多容器；

4.1安装

  • 下载
#github版本
curl -L "https://github.com/docker/compose/releases/download/v2.2.2/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose 
#国内高速版本
curl -L https://get.daocloud.io/docker/compose/releases/download/v2.4.1/docker-compose-`uname -s`-`uname -m` > /usr/local/bin/docker-compos
  • 加执行权限
chmod u+x /usr/local/bin/docker-compose
  • 查看版本
docker-compose --version
  • 卸载
rm /usr/local/bin/docker-cpmpose

4.2命令

docker-compose -h #查看帮助
docker-compose up #启动所有docker-compose
*docker-compose up -d #后台启动所有docker-compose
*docker-compose down #停止并删除容器、网络、卷、镜像
docker-compose exec yml里面的服务id #进入容器
docker-compose ps   # 展示当前docker-compose编排过的所有容器
docker-compose top  #展示当前docker-compose编排过的容器进程
docker-compose logs yml里的服务di #查看日志
docker-compose config #检查配置
*docker-compose config -q #有问题才有输出，检查docker-compose文件格式语法是否有错误
docker-compose restart #重启
docker-compose start   #启动
docker-compose stop    #停止

##参数说明
 
version: '3'					#compose编排版本为3'
services:						'//服务:'
  nginx:						'//服务名称'
    images: nginx:1.10			'//镜像'	
    hostname: nginx				'//容器主机名'
    container_name: nginx		'//容器名'
    ports:						'//提供端口:'
      - 1216:80					'//对外提供（暴露）的端口'
      - 1217:443
    networks:					'//指定网络'
      - test					'//网络名称（群集名称需一致）'
    volumes:					'//数据卷'
      - ./wwwroot:/usr/local/nginx/html		'//将宿主机该仓库内文件与nginx首页文件相关联（容器）'
   myapp:
     container_name: app
     ports:
       - 8080:8080
     build: #启动服务时，先将build中指定的dockerfile打包成镜像，再运行该镜像
       context: jenkins-demo #指定上下文目录dockerfile所在目录[相对、绝对路径都可以]
       dockerfile: Dockerfile #文件名称[在指定的context的目录下指定那个Dockerfile文件名称]
     networks:					
       - test		
     depends_on:
       - nginx    #指定先让nginx和mysql容器运行在运行app容器
       - mysql
    mysql:
      images: mysql:5.7
      environment:
        MYSQL_ROOT_PASSWORD: '123456'
      ports:
        - 3306:3306
      volumes:
        - /mydata/mysql/db:/var/lib/mysql
      networks:
        - test
      command: --default-authentication-plugin=mysql_native_password #解决外部无法访问
networks:		'//对外公开需要声明networks群集名称'
  test:
