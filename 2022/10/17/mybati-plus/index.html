1.Wrapper

用于sql中where的查询条件

实例

    @Test
    public void QueryWrapper()&#123;
        QueryWrapper<Student> wrapper = new QueryWrapper<>();
//        组装条件
        Map<String,Object> map=new HashMap<>();
        //map<key,value> key列名，value查询的值
        map.put("name","吕飞扬");
        map.put("age","22");
        wrapper.allEq(map);
        //调用map自己的查询方法
        List<Student> students = studentMapper.selectList(wrapper);
        for (Student student : students) &#123;
            System.out.println(student);

其实就是和Mybatis逆向工程差不多的东西，Wrapper是一个工具，用于封装条件，然后使用MP自带的查询，把Wrapper作为参数传进去就可以了

2.方法解读

2.1）select篇：QueryWrapper

  • allEq()：基于map的相等
    • 参数map（），用于封装查询条件
    • 如果Map对象中有key的value是null，则需要传入第二个参数boolean
      • 这个参数是用于判断是否考察是否要将null作为条件
      • true：处理空值，where条件加入字段is null
      • false：忽略空值,不作为where条件
  • last:拼接sql语句
    • 可以在QueryWrapper调用方法之后在调用last(SQL语句片段)
  • eq()：=
    • 参数（“列名”，“值”）
    • 查询条件相等的
  • ne()：！=
    • 参数（“列名”，“值”）
    • 查询条件不相等的
  • gt(): >
    • 参数（“列名”，“值”）
  • ge(): >=
    • 参数（“列名”，“值”）
  • lt(): <
    • 参数（“列名”，“值”）
  • le(): <=
    • 参数（“列名”，“值”）
  • between(): 在（？ and ？）
    • 参数（“列名”，“初始值”，“结束值”）
    • 包含等于
    • where 列名>=初始值 and 列名<=结束值
  • nobetween(): 不在（？and？）
    • 参数（“列名”，“初始值”，“结束值”）
    • where 列名<初始值 and 列名>结束值
  • like()：匹配值
    • 参数（“列名”，“值”）
    • like ‘%值%’
  • notlike()：不匹配值
    • 参数（“列名”，“值”）
    • not like ‘%值%’
  • likeLeft
    • like ‘%值’
  • likerRight
    • like ‘值%’
  • is null
    • 参数（“列名”）
  • is not null
    • 参数（“列名”）
  • in
    • 参数（“列名”，“值”，“值”，…..）
  • not in
    • 参数（“列名”，“值”，“值”，…….）
  • inSql：子查询
    • 参数（“列名”，“sql语句”）
    • where 列名 in sql语句结果
  • notInSql：子查询
    • 参数（“列名”，“sql语句”）
    • where 列名 not in sql语句结果
  • groupBy：自己写吧，没必要用MP框架写这个
  • OderByAsc：按字段升序
    • 参数（“列名”，……）
  • OderByDesc：按字段降序
    • 参数（“列名”，……）
  • orderBy()：
    • 参数
      • boolean condition：条件内容是否加入到sql语句的后面。
      • boolean isAsc：升序true 降序 false
      • colums：“列名”
  • exists：拼接sql

    • 参数（“SQL语句”）

    • 实例

      qw.exists("select * from student where age > 20")
      • 判断子查询条件是否成立
  • not exists: 和上面的相反
