redisson分布式锁

java 分布式 nosql redis
java nosql redis
发布日期:   2022-10-29

1.分布式锁

@Bean
 public Redisson redisson(){
     //单机配置
     Config config = new Config();
     config.useSingleServer().setAddress("redis://localhost:6379").setDatabase(0);
     return (Redisson) Redisson.create(config);
 }
      <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-data-redis</artifactId>
        </dependency>
<!--        分布式锁-->
        <dependency>
            <groupId>org.redisson</groupId>
            <artifactId>redisson</artifactId>
            <version>3.6.5</version>
        </dependency>

使用


@Resource
private Redisson redisson;

String key = xxx
Rlock lock = redisson.getLock(key)

//上锁
lock.lock()
//解锁
lock.unlock()

2.读写锁

  • 保证一定能读取到最新的数据，修改期间，写锁是一个互斥锁，读锁是一个共享锁
    • 读+读：相当于无锁，并发读
    • 写+读：等待写锁释放，在执行读锁
    • 写+写：阻塞
    • 读+写：等待读锁释放，在执行写锁
RReadWriteLock lock = redisson.getReadWriteLock("锁名")

//写锁
Rlock lock = lock.writeLock();

//读锁
Rlock lock = lock.readLock();

3.信号量锁

  • 指定一个信号量锁，然后自己赋值，只有获取到有设定的值不为0的情况才能执行业务
  • 比如卖东西使用这个锁就不错
RSeaphore park = redisson.getSemaphore("指定key");
//尝试获取信号量锁（信号量锁-1）
boolean b = park.tryAcquire()
if(b){
//业务逻辑
	xxx
}


//信号量锁+1
RSeaphore park = redisson.getSemaphore("指定key");
//释放一个锁
park.release()

4.闭锁

  • 指定一个数，然后等待，等这个数的锁全部释放完毕执行业务
  • 用于等待别的业务全部执行完在执行当前业务的场景
RCountDownLatch lock = redisson.getCountDownLatch("闭锁名")

lock.trySetCount(5);
lock.await();//等待闭锁都完成


RCountDownLatch lock = redisson.getCountDownLatch("闭锁名")
lock.countDown();//计数减1
