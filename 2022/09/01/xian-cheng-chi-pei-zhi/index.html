java线程池

java 流式编程 线程池
java
发布日期:   2022-09-01

1.JDK自带的ThreadPoolExecutor线程池七大参数

corePoolSize：核心线程数（一直存在，除非allowCoreThreadTimeOut）线程池创建号以后就准备就绪的线程

maxmumPoolSize：最大线程数，控制资源

keepAliveTime：存活时间，如果当前线程数大于核心线程数，将会对空闲的线程进行回收（核心线程一直存在，不回收）

unit：时间单位

BlockingQueue workQueue：阻塞队列。如果任务有很多，就会将多的任务放进队列里面，只要有线程空闲，就会去队列里面取出新的任务进行执行

threadFactory：线程的创建工厂

RejectedExecutionHandler handler：如果队列满了，按照指定的拒绝策略执行任务

1.1）4种拒绝策略

  • AbortPolicy - 抛出异常，中止任务。抛出拒绝执行 RejectedExecutionException 异常信息。线程池默认的拒绝策略。必须处理好抛出的异常，否则会打断当前的执行流程，影响后续的任务执行
  • CallerRunsPolicy - 使用调用者线程执行任务。当触发拒绝策略，只要线程池没有关闭的话，则使用调用者线程直接运行任务。一般并发比较小，性能要求不高，不允许失败。但是，由于调用者自己运行任务，如果任务提交速度过快，可能导致程序阻塞，性能效率上必然的损失较大
  • DiscardPolicy - 直接丢弃，其他啥都没有
  • DiscardOldestPolicy - 丢弃队列最老任务，添加新任务。当触发拒绝策略，只要线程池没有关闭的话，丢弃阻塞队列 workQueue 中最老的一个任务，并将新任务加入

1.2）3种常用队列

  • 直接提交队列：设置为SynchronousQueue队列，SynchronousQueue是一个特殊的BlockingQueue，它没有容量，每执行一个插入操作就会阻塞。

    • 提交的任务会被马上执行，并且执行完后当前线程阻塞，不会复用，所以当创建的线程数大于maximumPoolSize时，直接执行了拒绝策略。

  • 有界的任务队列：设置为ArrayBlockingQueue(队列数) 队列

    • 若有新的任务需要执行时，线程池会创建新的线程，直到创建的线程数量达到核心线程数（corePoolSize）时，会将新的任务加入到等待队列中。若等待队列已满，即超过ArrayBlockingQueue初始化的容量，则继续创建线程，直到线程数量达到最大线程数（maximumPoolSize）设置的数量，若还有任务进入，则执行拒绝策略。

  • 无界的任务队列：设置为LinkedBlockingQueue队列

    • 线程池的任务队列可以无限制添加新任务，而线程池创建的最大线程数就是核心线程数(corePoolSize)设置的数量，这种情况下maximumPoolSize设置的值是不会生效的，就算等待队列中有许多未执行的任务，线程池的数量达到了corePoolSize的值后也不会增加新的线程。

2.运行流程

  1. 线程池创建，准备core数量的核心线程，准备接收任务
  2. 新的任务进来，用core准备好的空闲线程执行
    1. core满了，将多的任务放入阻塞队列中
    2. 阻塞队列满了，直接开启新的线程执行，但数量不超过指定的最大线程数
      • 如果执行完毕，空闲线程会在keepAliveTIme指定的时间销毁，最终保持到只剩core（核心线程数）大小
      • 如果线程数开到了指定的最大线程数，还有新的任务进入，就会使用指定的RejectedExecutionHandler （拒绝策略）进行处理
  3. 所有的线程都是由指定的factory创建的

问：一个线程池 core 7 、max 20、queue 50、100并发进来如何分配

答：先有7个能直接被执行，然后50个进入阻塞队列，在多开13个线程执行，剩下30个执行拒绝策略

3.CompletableFuture异步编排

3.1）whenComplete()感知异常，exceptionally()处理异常

public class Test {
    private static ThreadPoolExecutor executor =  new ThreadPoolExecutor(5,
            20,
            60,
            TimeUnit.SECONDS,
            new LinkedBlockingQueue<>(),
            new ThreadPoolExecutor.DiscardPolicy() );


    public static void main(String[] args) throws ExecutionException, InterruptedException {
        CompletableFuture<Integer>  future = CompletableFuture.supplyAsync(()->{
            System.out.println("当前线程"+Thread.currentThread().getId());
            int i = 10/0;
            return i;
        },executor).whenComplete((res,exception)->{
            //whenComplete可以感知异常但没法处理
            System.out.println("异步线程执行完毕，结果是"+res+"异常是"+exception);
            
        }).exceptionally(throwable -> {
              //exceptionally 出现异常时执行此方法
               return 10;
            });

        System.out.println("结果"+future.get());
    }
}
  • whenComplete：
    • 参数：结果，异常
    • 可以感知异常，但是没法处理异常
  • exceptionally：出现异常时进入方法处理结果

3.2）handler()处理异常

public static void main(String[] args) throws ExecutionException, InterruptedException {
    CompletableFuture<Integer>  future = CompletableFuture.supplyAsync(()->{
        System.out.println("当前线程"+Thread.currentThread().getId());
        int i = 10/2;
        return i;
    },executor).handle((result,exception)->{
        if(null != exception){
            return 10*2;
        }
        return result;
    });
    System.out.println("结果"+future.get());
}
  • handler：
    • 参数：结果，异常
    • 优点：简洁，如果出现异常，可以直接进行处理

3.3）异步编排

首先，有两个任务，其中一个立即执行，另一个沉睡3秒

      //任务1
CompletableFuture<Boolean>  future1 = CompletableFuture.supplyAsync(()->{
          try {
              Thread.sleep(3000);
              System.out.println("future1执行完毕");
          } catch (InterruptedException e) {
              e.printStackTrace();
          }
          return true;
      },executor).exceptionally(throwable -> {
          System.out.println("出现异常");
          return false;
      });

//任务2
      CompletableFuture<Boolean>  future2 = CompletableFuture.supplyAsync(()->{
              System.out.println("future2执行完毕");
          return true;
      },executor).exceptionally(throwable -> {
          System.out.println("future2出现异常");
          return false;
      });

3.3.0）等待上一个任务执行完毕在执行

//future1：上一个任务，res形参：上一个任务的结果返回值
future1.thenAcceptAsync(res->{
	//xxx 业务逻辑
},excutor)

3.3.1）两个异步任务执行完才执行方法

    /**
     * 执行结果：
     * 
     * future2执行完毕
     * future1执行完毕
     * 全部执行完毕
     */
任务1.thenAcceptBothAsync(任务2,(任务1结果，任务2结果)->{
        //逻辑
    },线程池)
   任务1.thenCombineAsync(任务2,(任务1结果，任务2结果)->{
        //逻辑
       //return 返回值
    },线程池)

thenAcceptBothAsync：没有返回值，当两个任务执行完毕，执行此方法

//任务1和任务2都执行完毕进入此方法
      future1.thenAcceptBothAsync(future2,(f1,f2)->{
          System.out.println("全部执行完毕");
      },executor);
  }
  • thenCombineAsync：有返回值，当两个任务执行完毕，执行此方法
CompletableFuture<String> future = future1.thenCombineAsync(future2, (f1, f2) -> {
    return "全部任务执行完毕";
}, executor);
System.out.println(future.get());

3.3.2）两个任务任意执行完一个即可执行方法

 /**
  * 执行结果：
  *
  * future2执行完毕
  * 任意一个任务执行完毕，即可执行此方法
  * future1执行完毕
  */
任务1.applyToEitherAsync(任务2,(最先执行完毕的任务结果)->{
     //逻辑
    //return 返回值
 },线程池)

future1.runAfterEitherAsync(future2,()->{
    System.out.println("任意一个任务执行完毕，即可执行此方法");
},executor);
CompletableFuture<String> future3 = future1.applyToEitherAsync(future2, res -> {
    return "执行完毕任务3";
}, executor);
System.out.println(future3.get());

runAfterEitherAsync()：不感知结果，自己没有返回值

applyToEitherAsync()：感知结果，自己有返回值

3.3.3）多个任务执行完毕后才执行方法

CompletableFuture<Void> allOf = CompletableFuture.allOf(future1, future2);
allOf.get();
System.out.println("执行完毕");

3.3.4）多个任务任一执行完毕后执行

CompletableFuture<Object> future = CompletableFuture.anyOf(future1, future2);
future.get();
System.out.println("有一个执行完了");

4.Spring自带的线程池

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.scheduling.annotation.EnableAsync;
import org.springframework.scheduling.concurrent.ThreadPoolTaskExecutor;

import java.util.concurrent.Executor;
/**
 * 线程池初始化
 *
 * @author lvfeiyang
 * @email 1316547530@qq.com
 * @date 2022-7-02 19:02:04
 */
@Configuration
@EnableAsync // 开启线程池
public class ThreadPoolConfig {

    @Bean("taskExecutor")
    public Executor asyncServiceExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();

        // 设置核兴线程数
        executor.setCorePoolSize(5);
        // 设置最大线程数
        executor.setMaxPoolSize(20);
        // 设置队列大小
        executor.setQueueCapacity(Integer.MAX_VALUE);
        // 设置线程活跃时间 60s
        executor.setKeepAliveSeconds(60);
        // 设置默认线程名称
        executor.setThreadNamePrefix("lfy");
        // 是否所有任务执行完毕后关闭线程池
        executor.setWaitForTasksToCompleteOnShutdown(true);
        // 执行初始化
        executor.initialize();
        return executor;
    }
}
